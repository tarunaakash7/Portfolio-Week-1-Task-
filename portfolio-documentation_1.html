🎯Project Overview

🚀Setup Instructions

Prerequisites

  • Git - https://git-scm.com/
  • Node.js (v14 or higher) - https://nodejs.org/
  • npm - Comes with Node.js
  • Code Editor - VS Code recommended

Step 1: Clone Repository

git clone https://github.com/yourusername/portfolio.git cd portfolio

Step 2: Install Dependencies

npm install

Step 3: Environment Setup

Create a .env file in root directory:

REACT_APP_EMAILJS_SERVICE_ID=your_service_id REACT_APP_EMAILJS_TEMPLATE_ID=your_template_id REACT_APP_EMAILJS_USER_ID=your_user_id

Step 4: Run Locally

npm start

Opens http://localhost:3000 automatically

Step 5: Build for Production

npm run build
✓ Success! Your portfolio is now running locally. Make changes to see them in real-time.

🌐Deployment Guide

Option 1: Netlify (Recommended)

  1. Push your repository to GitHub
  2. Visit https://netlify.com
  3. Click "New site from Git"
  4. Select your repository
  5. Build command: npm run build
  6. Publish directory: build
  7. Click "Deploy"

Option 2: Vercel

  1. Go to https://vercel.com
  2. Import your GitHub repository
  3. Click "Deploy"
  4. Your site will be live instantly

Option 3: GitHub Pages

npm install --save-dev gh-pages # Add to package.json: # "homepage": "https://yourusername.github.io/portfolio" # "deploy": "npm run build && gh-pages -d build" npm run deploy
✓ Go Live! Your portfolio is now accessible online. Share your URL with recruiters and clients!

🐛Troubleshooting

Issue: Website not loading Solution:

✓ Check if all file paths are correct ✓ Ensure CSS and JS files are in the correct directories ✓ Open browser console (F12) to check for errors

Issue: Contact form not working Solution:

✓ Verify EmailJS credentials in .env file ✓ Check email service configuration ✓ Test with console.log statements

Issue: Images not displaying Solution:

✓ Verify image paths are relative or absolute URLs ✓ Ensure image files exist in the images folder ✓ Check file permissions

Issue: Responsive design not working Solution:

✓ Add viewport meta tag: <meta name="viewport"> ✓ Check CSS media queries ✓ Test in browser DevTools (F12)

Issue: Slow loading on mobile Solution:

✓ Compress images using TinyPNG ✓ Minimize CSS and JavaScript files ✓ Use lazy loading for images ✓ Enable caching in deployment settings
